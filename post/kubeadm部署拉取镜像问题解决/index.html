kubeadm拉取谷歌镜像被墙问题解决

2019-03-10
Kubernetes

Table of Contents

    • 需要使用到的工具及平台
    • 解决步骤

很多同学使用kubeadm安装kubernetes集群最大的拦路虎可能就是几个k8s.gcr.io镜像的下载了，比较肉的方法可能是去docker-hub上搜索相关版本镜像pull下来，然后重新tag下。其实docker-hub上面也是别人从国外pull下来的重新上传上去了，可能很多版本要找对应的比较费劲。

不扯淡了，直接说方法

需要使用到的工具及平台

  • 阿里云容器镜像仓库（可以海外构建）
  • github（提交dockerfile，触发自动构建）
  • dockerfile文件

解决步骤

只提供解决思路

1、查看当前kubeadm部署的镜像版本

$ kubeadm config images list
I0508 13:47:27.777458   11521 version.go:96] could not fetch a Kubernetes version from the internet: unable to get URL "https://dl.k8s.io/release/stable-1.txt": Get https://dl.k8s.io/release/stable-1.txt: net/http: request canceled while waiting for connection (Client.Timeout exceeded while awaiting headers)
I0508 13:47:27.777532   11521 version.go:97] falling back to the local client version: v1.14.1
k8s.gcr.io/kube-apiserver:v1.14.1
k8s.gcr.io/kube-controller-manager:v1.14.1
k8s.gcr.io/kube-scheduler:v1.14.1
k8s.gcr.io/kube-proxy:v1.14.1
k8s.gcr.io/pause:3.1
k8s.gcr.io/etcd:3.3.10
k8s.gcr.io/coredns:1.3.1

2、dockerfile

目录 创建好对应的目录，一个镜像一个仓库一个目录

sudo mkdir /opt/kuberentes_containers/{etcd,kube-apiserver,kube-controller-manager,kube-scheduler,kube-proxy,pause,coredns}

dockerfile

# etcd/Dockerfile
FROM k8s.gcr.io/etcd:3.3.10

# kube-apiserver/Dockerfile
FROM k8s.gcr.io/kube-apiserver:v1.14.1

# kube-controller-manager/Dockerfile
FROM k8s.gcr.io/kube-controller-manager:v1.14.1

# kube-scheduler/Dockerfile
FROM k8s.gcr.io/kube-scheduler:v1.14.1

# pause/Dockerfile
FROM k8s.gcr.io/pause:3.1

# kube-proxy/Dockerfile
FROM k8s.gcr.io/kube-proxy:v1.14.1

# coredns/Dockerfile
FROM k8s.gcr.io/coredns:1.3.1

将目录提交的github,后面更新版本只需要更新下dockerfile就可以

3、阿里云容器镜像仓库使用

  • 创建空间，公开的
  • 一个镜像一个仓库
  • 自动构建关联github
  • 构建时选择使用海外构建

都是界面配置，不细说，一看就会

4、kubeadm 使用指定的镜像仓库

kubeadm init \
--kubernetes-version=v1.14.1 \
--pod-network-cidr=10.244.0.0/16 \
--apiserver-advertise-address=192.168.2.11 \
--image-repository=registry.cn-hangzhou.aliyuncs.com/google_con # 你自己的仓库地址就可以
